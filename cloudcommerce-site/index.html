Why this exists

CloudCommerce isn’t a real shop — it’s the working front end for an AWS Solutions Architect capstone. It’s intentionally minimal so the interesting work is the infrastructure behind it: global delivery, a containerised API, purpose-built databases, an event-driven order pipeline, and a CI/CD pipeline that ships it all safely.

Start with the catalog you see here as static files in S3. By the end of the capstone, every “Add to cart” and “Checkout” will travel through a production-grade, multi-AZ architecture.

Backend connection

Running in static mode — products are served from data/products.json. Set API_BASE in assets/config.js to point at your ALB once Lab 3 is done.
